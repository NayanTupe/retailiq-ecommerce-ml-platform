# RetailIQ API Documentation

## Project Name

RetailIQ: E-commerce Customer Churn Prediction Platform

---

## API Overview

RetailIQ API is built using FastAPI. It provides backend endpoints for application health checks, customer churn prediction, dashboard summary analytics, customer segmentation analytics, and churn analytics by membership type.

This API is designed so that a future React or Next.js dashboard can consume machine learning and analytics data from the backend.

---

## Base URL

For local development:

```text
http://127.0.0.1:8000
```

Swagger API documentation:

```text
http://127.0.0.1:8000/docs
```

---

* FastAPI

* Pydantic

## Available Endpoints

| Method | Endpoint                         | Purpose                           |
| ------ | -------------------------------- | --------------------------------- |
| GET    | `/`                              | API root status                   |
| GET    | `/health`                        | Health check                      |
| POST   | `/predict/churn`                 | Predict customer churn risk       |
| GET    | `/analytics/summary`             | Get dashboard KPI summary         |
| GET    | `/analytics/segments`            | Get customer segment analytics    |
| GET    | `/analytics/churn-by-membership` | Get churn rate by membership type |

# 3. Churn Prediction Endpoint

## Endpoint

```http
POST /predict/churn
```

## Purpose

Predicts whether a customer is likely to churn and returns a business-friendly result.

The response includes:

* Churn prediction
* Churn probability
* Risk level
* Business recommendation

---

## Request Body

```json
{
  "gender": "Female",
  "age": 32,
  "city": "Miami",
  "membership_type": "Bronze",
  "total_spend": 166.56,
  "items_purchased": 2,
  "average_rating": 2.5,
  "discount_applied": true,
  "avg_spend_per_item": 83.28,
  "is_high_value_customer": 0,
  "is_frequent_buyer": 0,
  "low_rating_flag": 1,
  "discount_used_flag": 1
}
```

---

## Response Body

```json
{
  "churn_prediction": 1,
  "churn_probability": 0.8414,
  "risk_level": "High Risk",
  "recommendation": "Offer retention discount, loyalty reward, or personal follow-up."
}
```

---

## Response Field Meaning

| Field               | Meaning                                     |
| ------------------- | ------------------------------------------- |
| `churn_prediction`  | 1 means churn risk, 0 means no churn risk   |
| `churn_probability` | Probability score generated by the ML model |
| `risk_level`        | Business-friendly risk category             |
| `recommendation`    | Suggested action for the business owner     |

## How to Run API Locally

Activate virtual environment:

```bash
source venv/bin/activate
```

Run FastAPI server:

```bash
python -m uvicorn api.main:app --reload
```

Open Swagger documentation:

```text
http://127.0.0.1:8000/docs
```

## React Integration Plan

In the future React dashboard, these endpoints can be consumed like this:

```text
React Dashboard
    ↓
FastAPI Backend
    ↓
ML Model / Processed Analytics Data
    ↓
JSON Response
    ↓
Charts, Cards, Tables, Prediction UI
```

Example React API usage:

```js
const response = await fetch("http://127.0.0.1:8000/analytics/summary");
const data = await response.json();
```

```js

    gender: "Female",
```

## Notes

Generated model files and processed datasets are not pushed to GitHub because they are ignored using `.gitignore`.

Ignored generated files include:

```text
models/
data/raw/
data/interim/
data/processed/
```

To run the API successfully, the model and processed data should be generated locally by running the data pipeline and training scripts.
